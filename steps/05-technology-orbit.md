# Step 05 — Technology Orbit

**Purpose:** Show skills and tools as orbiting particles around SV core.  
**Last Updated:** 20 May 2026, 15:40 IST

---

## Scene goal

The user inspects technical capability.

Only skills, technologies, and selected tools belong in this orbit.

---

## Orbit groups

```txt
Inner Orbit:
HTML5, CSS3, JavaScript, TypeScript

Middle Orbit:
React, Next.js, Node.js, Express.js, PHP, WordPress

Outer Orbit:
MongoDB, MySQL, Tailwind CSS, Bootstrap, Sass / SCSS, jQuery, Redux, JWT Auth

Deep Zoom Orbit:
WooCommerce, Git, Figma, GSAP, Three.js, APIs, Performance, Hosting, Analytics, Search Console
```

---

## Do not include

Never include these in Technology Orbit:

- product names,
- reviews,
- proof numbers,
- contact actions,
- service paragraphs.

Those belong to separate scenes.

---

## Particle behavior

Default:

- cyan glow,
- short readable label,
- slow orbit.

Hover desktop:

- orange/yellow lock-on glow,
- scan ring,
- label brightens,
- rocket reticle reacts.

Tap/click:

- cyan ripple,
- detail panel opens,
- selected particle stays locked.

---

## Detail panel

Desktop:

- compact floating black glass panel,
- short content,
- no article padding,
- never blocks core.

Mobile:

- bottom sheet,
- readable,
- close button,
- does not cover bottom controls.

---

## Zoom behavior

- deeper tools reveal only at deeper zoom.
- zoom never breaks layout.
- labels stay readable.

---

## Scene transition

Previous: Arrival Core.  
Next: Product Gallery.

When leaving:

- skill particles fade or move backward,
- selected detail closes,
- product beacons become active.

---

## Completion checklist

- [ ] Only skills/tools orbit the core.
- [ ] Labels are short.
- [ ] Hover and tap feedback work.
- [ ] Detail panel opens and closes.
- [ ] Mobile bottom sheet works.

# Expert Flow Management Refinement

## Scene Identity

```txt
Scene: Technology Orbit
Previous: Arrival Core

```

## Connected Layers / Cards / Objects

Connected layers:

- SV Core
- Orbit Layer
- Skill Particles
- Skill Detail Sheet
- Zoom System

Card/panel rule:

```txt
Use particle detail sheet only. Do not use website cards.
```

## Animation by Device

### Mobile 375px

```txt
One selected skill focus. Tap opens bottom sheet. Minimal orbit movement.
```

### Tablet 768px

```txt
Particles reveal in rings. Tap/hover detail panel with moderate motion.
```

### Desktop 1366px+

```txt
Orbit rings reveal, hover lock-on, scan ring, rocket reticle reaction.
```

## Transition Rules

Before entering this scene:

1. Close any panel that does not belong to this scene.
2. Preserve the SV core safe zone.
3. Update the active bottom step/navigation state.
4. Reveal only this scene’s main focus.
5. Keep unrelated scene layers hidden.

Before exiting this scene:

1. Close open details unless the next scene needs them.
2. Fade or recede current scene elements.
3. Trigger the next scene’s reveal motion.
4. Do not show both previous and next scene content at full attention.

## Failure Risks to Avoid

- Products/reviews/proof mixed into orbit
- Too many labels visible
- Detail panel covers core
- Labels too long
