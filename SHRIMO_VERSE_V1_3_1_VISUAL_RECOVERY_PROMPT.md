# Shrimo Verse v1.3.1 Visual Recovery Prompt

Target output: `shrimo-verse-v1.3.1-visual-recovery.zip`

## Core Direction

Shrimo Verse is not a normal website, article page, SaaS page, dashboard, or portfolio layout. It is a cinematic black/cyan universe experience.

The goal of this version is to recover the visual experience after v1.3 became too layered, bordered, and website-like.

## Rules Followed

1. Keep the static HTML/CSS/JS architecture.
2. Do not rebuild or convert to React.
3. Remove dashboard-like default panels.
4. Use black/cyan as the main visual system.
5. Do not use orange/yellow for selected states or controls.
6. Reduce borders, padding, and article-card styling.
7. Keep one scene focus at a time.
8. Hide debug/status information from the default cinematic view.
9. Make mobile the primary view.
10. Keep `README.md` and `about-project.md` in the root.

## Specific Visual Fixes

- Hide `Live Build Deck` by default.
- Hide zoom dock by default.
- Turn the mission panel into a lightweight scene caption.
- Remove internal scrolling from the main scene caption.
- Reduce the large cyan background blob into subtle atmosphere.
- Simplify the mobile bottom HUD.
- Convert Launch Dock into a clean conversion signal, not an article card.
- Keep WhatsApp as the primary CTA but style it with black/cyan.
- Show call/email only as secondary actions on mobile.
- Keep confidence signals as simple text, not bordered cards.
- Make guide/settings/briefing sheets compact and non-competing.

## Validation Rule

Do not call the build final unless scripts load, CSS imports exist, duplicate IDs are checked, required docs are present, and ZIP integrity passes.

Real mobile QA and Lighthouse scores must not be claimed unless tested on actual devices.
